# OctoAcme Project Management — Process Index

This folder centralizes OctoAcme's project management processes and serves as the canonical entry point for onboarding, reference, and contributions.

## Summary
- OctoAcme runs projects with a lightweight, iterative approach focused on customer value, clear ownership, measurable outcomes, and continuous improvement.
- These documents cover the full lifecycle: Initiation → Planning → Execution → Release → Retrospective.
- Use this README to find phase-specific guidance, quick references for roles/artifacts, and instructions for contributing improvements.

## Core principles
- Customer-first: prioritize impact and usability.
- Iterative delivery: small, testable increments.
- Clear ownership: each project has a named PM and Product Lead.
- Data-informed decisions: measure outcomes and iterate.
- Psychological safety: encourage learning and feedback.

## Quick role summaries
- Project Manager (PM): coordinates delivery, schedules, risks, and communications.
- Product Manager (PdM): defines outcomes, prioritizes backlog, and measures success.
- Developers: build, test, and deliver features.
- QA/Testing: validate acceptance criteria and quality.

## Communication cadence (quick reference)
- Daily: standups (15 min) for progress/blockers.
- Weekly: delivery sync and PM/PdM alignment.
- Sprint/milestone: demo/review at end of iteration.
- Monthly: stakeholder summary (high-level progress & risks).
- Ad-hoc: escalate based on the documented escalation paths.

## How to use these docs
- New starters: read the Project Management Overview, then the phase-specific docs for your role.
- Project owners: keep your project one-pager and release plan in the project repo and link to these process docs as needed.
- Copilot Spaces: put project-specific process variants into `.copilot/` or docs/ in the project repository to make them available as contextual guidance.
